# Redis (кешування та сесії)

У цьому проєкті Redis використовується як кеш та швидке сховище ключ-значення. Redis може допомогти зменшити навантаження на базу даних, зберігати сесії, публікації/підписки (pub/sub), або як брокер для тасків.

## Чому Redis
- Швидке зберігання в пам'яті для часто читаних даних
- Може використовуватися для обробки сесій та кешування результатів запитів
- Підтримка pub/sub і структур даних (списки, множини, хеші, сортування)

## Встановлення Redis

див файл `redis_windows_install.docx` для Windows

### Налаштування в проєкті
Проєкт зчитує налаштування з .env через `settings.py`. Параметри, які можна задати:

- `REDIS_HOST` (за замовчуванням: `localhost`)
- `REDIS_PORT` (за замовчуванням: `6379`)
- `REDIS_DB` (за замовчуванням: `0`)
- `REDIS_PASSWORD` (за замовчуванням: `None`)


### Тест підключення
Для швидкої перевірки підключення до Redis використовуйте `ping_redis.py`:

```bash
python ping_redis.py
```

Очікуваний вивід:

```
✅ Redis OK!
```

### Використання Redis у коді
У `settings.py` вже є метод для створення асинхронного клієнта Redis:

```python
from settings import api_config

redis = api_config.redis_client()
await redis.set('key', 'value')
value = await redis.get('key')
```
